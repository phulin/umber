# PDF Tagged Spacing and Accessibility

This document defines Umber's implementation boundary for pdfTeX 1.40.27's
fake-interword-space controls. Durable task status lives in Beads under
`umber2-kbz0.18`; this is the technical design and verification contract.

## Reference behavior

The pdfTeX manual and `texk/web2c/pdftexdir/pdftex.web` define three ordered
whatsits: `\pdfinterwordspaceon`, `\pdfinterwordspaceoff`, and
`\pdffakespace`. The on/off state is interpreted while a shipped list is
traversed and therefore crosses font, box, and page boundaries in output
order. A fake-space node always emits a real slot-32 space without changing
TeX's horizontal dimensions.

With interword spaces enabled, pdfTeX recognizes qualifying horizontal motion
before the next character. If that character's font has an explicit encoding
whose slot 32 is named `/space`, it emits the space from that font. Otherwise
it lazily loads the fallback selected by `\pdfspacefont`, initially
`pdftexspace`. The following movement is compensated so painting remains at
the position selected by TeX.

`\pdfspacefont` scans expanded general text, acts globally even inside a TeX
group, and only affects the fallback before its first use. All four primitives
are errors when `\pdfoutput <= 0`.

The character controls `\tagcode`, `\knbscode`, `\stbscode`, `\shbscode`,
`\knbccode`, and `\knaccode` already have checkpointed state and execution
semantics. This work reuses those font identities and does not introduce a
second character-code store.

The unexpandable meaning operands for this slice are `247..=250`. Operands
`238..=246` are reserved by the accepted PDF form/image slice developed in
parallel; they must not be reused while that dependency awaits canonical
integration.

## Architecture

Execution appends typed accessibility whatsits after flushing pending
characters. Shipout lowers them to typed, versioned page effects; DVI ignores
those PDF-only effects. The PDF positioned traversal exposes each effect at
its exact coordinate and retains the TeX coordinate of every potential space
and following text segment.

PDF assembly carries one interword-space state across committed pages. It
splits text at potential spaces, omits the space while disabled, and emits it
from either the current encoded font or one lazily created fallback resource
while enabled. Each following segment receives an absolute text matrix, so
the accessible character cannot perturb painted geometry. `\pdffakespace`
uses that fallback path unconditionally.

All generated content operators and fallback font objects pass through the
typed `pdf_writer` adapter. Raw PDF fragments remain limited to user-authored
pdfTeX escape hatches already modeled by the backend.

The selected fallback font name is checkpointed in `PdfState` and captured in
each page receipt. This reproduces deferred output: assignments before a page
ships affect that page, while rollback restores the exact prior selection.
Once PDF assembly first needs the fallback, later page selections do not
replace the already loaded document font.

## Verification matrix

| Boundary      | Required evidence                                                                                                    |
| ------------- | -------------------------------------------------------------------------------------------------------------------- |
| Scanning      | expanded `\pdfspacefont` text, prefix rejection, global grouping, and DVI-mode errors match pdfTeX                   |
| State         | snapshots, semantic hashes, format emptiness, rollback, and replay include the fallback selection and typed whatsits |
| Artifact      | accessibility effects round-trip in the current schema; older supported schemas still decode                         |
| Operators     | off/on/off transitions and forced fake spaces have exact content-stream assertions generated by `pdf_writer`         |
| Fonts         | encoded slot-32 `/space` uses the current font; otherwise the selected fallback resource is reused document-wide     |
| Structure     | font switches, ligatures, nested boxes, and page breaks preserve ordered state and exact TeX anchors                 |
| Accessibility | Poppler extraction observes the inserted spaces while raster output remains unchanged                                |
| Regression    | focused crate tests, PDF parity fixtures, full workspace tests, rustfmt, and clippy pass                             |

The committed `pdf/embedded_tagged_spacing` corpus case is the pinned
pdfTeX 1.40.27/Poppler 25.08.0 oracle for this matrix. It records reference and
Umber structures separately because pdfTeX embeds its fallback Type-1 font
while Umber deliberately creates the equivalent invisible Type-3 `/space`
through `pdf_writer`; exact extraction and raster attestations remain shared.
